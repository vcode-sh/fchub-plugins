This takes about 5 minutes. You'll generate a password in WordPress, paste a config snippet into your AI client, and you're done. I've made it as boring and straightforward as possible — no ceremony, no hoops.

## Claude Desktop Extension

No terminal. No JSON. No Node.js. Just download a file, double-click it, fill in three fields.
If you can install a browser extension, you can do this.

<Steps>
<Step>
### Download the Extension

<Cards>
  <Card title="Download fluentcart-mcp.mcpb" href="https://github.com/vcode-sh/fchub-plugins/releases/download/fluentcart-mcp/v1.0.0/fluentcart-mcp.mcpb">
    Latest release — click to download.
  </Card>
</Cards>

Or browse all versions on the [GitHub Releases page](https://github.com/vcode-sh/fchub-plugins/releases).

</Step>
<Step>
### Install It

Double-click the `.mcpb` file. Claude Desktop picks it up automatically.

Or drag it into Claude Desktop. Or go to **Settings → Extensions → Install Extension** and select the file.
Three ways to do the same thing — pick whichever makes you feel most in control.

</Step>
<Step>
### Enter Your WordPress Credentials

Claude Desktop shows three fields:

| Field | What to enter |
|-------|---------------|
| **WordPress URL** | `https://your-store.com` — no trailing slash, no `/wp-admin/` |
| **Username** | Your WordPress admin username |
| **Application Password** | Generate one at **Users → Profile → Application Passwords** |

<Callout type="warn" title="WordPress Shows This Once">
The Application Password appears exactly once when you create it.
Close that page without copying and you're generating a new one. Ask me how I know.
</Callout>

</Step>
<Step>
### Use It

Open a new chat. The hammer icon in the input area should show FluentCart tools. Try:

> "Show me today's orders"

See store data? Done. No JSON was harmed in the process.

</Step>
</Steps>

or Codex CLI — keep reading. The developer setup below covers all of them.

## Prerequisites

<Steps>
<Step>
### Install Node.js 22+

The MCP server needs Node.js 22 or higher. Check what you've got:

```bash
node --version
```

See `v22.x.x` or higher? Brilliant. Move on. If not, grab it from [nodejs.org](https://nodejs.org) or use [nvm](https://github.com/nvm-sh/nvm):

```bash
nvm install 22
nvm use 22
```

<Callout type="info" title="Why 22?">
I use native fetch and top-level await. These aren't optional — they're baked into how the server works. Node 20 and below won't cut it.
</Callout>

</Step>
<Step>
### Generate a WordPress Application Password

Application Passwords have been built into WordPress since version 5.6. They let the MCP server talk to your site's REST API without you handing over your actual login password. Sensible, that.

1. Log in to your WordPress admin
2. Go to **Users → Profile** (or click your username in the top-right)
3. Scroll to the **Application Passwords** section
4. Type a name — `FluentCart MCP` works fine
5. Click **Add New Application Password**
6. **Copy the password immediately**

It looks something like: `aBcD eFgH iJkL mNoP qRsT uVwX`

<Callout type="warn" title="WordPress Shows This Once">
I mean it. WordPress shows the password exactly once. Close that page without copying it and you're generating a new one. Ask me how I know.
</Callout>

<Callout type="info" title="Which Account?">
Use an **Administrator** account. The MCP server needs admin-level access to manage orders, products, and settings. A limited-permission user will get 403 errors on nearly everything, which rather defeats the purpose.
</Callout>

</Step>
<Step>
### Grab Your Three Values

You need exactly these:

| Value | Example |
|-------|---------|
| **WordPress URL** | `https://your-store.com` |
| **Username** | `admin` |
| **Application Password** | `aBcD eFgH iJkL mNoP qRsT uVwX` |

The URL is the root of your WordPress site. No trailing slash. No `/wp-admin/`. Just the domain.

</Step>
</Steps>

## Configure Your AI Client

Pick your weapon. Each one takes about 60 seconds.

<Tabs items={['Claude Desktop', 'Claude Code', 'Cursor', 'VS Code + Copilot', 'Windsurf', 'Codex CLI', 'Docker (Remote)']}>
<Tab value="Claude Desktop">

### Claude Desktop

Anthropic's standalone app. The one with the nice icon.

<Callout type="info" title="Prefer the Easy Way?">
Skip all this. Use the [Claude Desktop Extension](#claude-desktop-extension) above instead —
one file, no terminal, no JSON editing.
</Callout>

<Steps>
<Step>
### Open the Config File

- **macOS**: **Claude** menu → **Settings** → **Developer** → **Edit Config**
- **Windows**: **File** → **Settings** → **Developer** → **Edit Config**

This opens `claude_desktop_config.json`. If you prefer to find it manually:
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`
</Step>

<Step>
### Paste This

Replace the placeholder values with your actual credentials:

```json
{
  "mcpServers": {
    "fluentcart": {
      "command": "npx",
      "args": ["-y", "fluentcart-mcp"],
      "env": {
        "FLUENTCART_URL": "https://your-store.com",
        "FLUENTCART_USERNAME": "admin",
        "FLUENTCART_APP_PASSWORD": "aBcD eFgH iJkL mNoP qRsT uVwX"
      }
    }
  }
}
```

<Callout type="info" title="Already Have Other MCP Servers?">
Don't replace the whole file. Add `"fluentcart": {...}` inside your existing `mcpServers` object. Comma after the previous entry. You know the drill.
</Callout>

</Step>

<Step>
### Restart the App

Fully close and reopen Claude Desktop. Look for the hammer icon in the chat input — click it and you should see FluentCart tools listed.

No hammer? Check **Settings → Developer** for error messages.
</Step>
</Steps>

### Claude Code

The terminal-based one. My personal favourite, obviously.

<Steps>
<Step>
### One Command

```bash
claude mcp add fluentcart \
  -e FLUENTCART_URL=https://your-store.com \
  -e FLUENTCART_USERNAME=admin \
  -e FLUENTCART_APP_PASSWORD="aBcD eFgH iJkL mNoP qRsT uVwX" \
  -- npx -y fluentcart-mcp
```

That's it. Start a new session and you're connected.

</Step>
<Step>
### Check It Works

In your next Claude Code session, try:

```
Show me the FluentCart dashboard stats
```

If you see store data, you're golden. You can also verify with:

```bash
claude mcp list
```

</Step>
</Steps>

<Tab value="Docker (Remote)">

### Docker — Remote / VPS / ChatGPT

If you want a persistent MCP endpoint that remote clients (ChatGPT, custom agents, other machines) can reach — run the Docker image. No Node.js install needed.

<Steps>
<Step>
### Pull and run

```bash
docker run -d \
  -p 3000:3000 \
  -e FLUENTCART_URL=https://your-store.com \
  -e FLUENTCART_USERNAME=admin \
  -e FLUENTCART_APP_PASSWORD="aBcD eFgH iJkL mNoP qRsT uVwX" \
  -e FLUENTCART_MCP_API_KEY=pick-a-strong-secret \
  --name fluentcart-mcp \
  vcodesh/fluentcart-mcp
```

That's it. The image runs the HTTP transport on port 3000 automatically.

</Step>
<Step>
### Verify

```bash
curl http://localhost:3000/health
```

Should return `{"status":"ok"}`. If it does, your MCP endpoint is live at `http://localhost:3000/mcp`.

</Step>
<Step>
### Connect your AI client

Point any HTTP-capable MCP client at your endpoint. For ChatGPT, paste the URL into the MCP server configuration. For remote access, put it behind a reverse proxy or Cloudflare Tunnel — the [Deployment Guide](/docs/fluentcart-mcp/deployment) walks you through it.

</Step>
</Steps>

<Callout type="warn" title="Set FLUENTCART_MCP_API_KEY">
The `FLUENTCART_MCP_API_KEY` protects the MCP endpoint with a bearer token. Without it, anyone who finds your URL gets full access to your store. I'm not your mum, but I am telling you to set it.
</Callout>

## Alternative: Shared Config File

If you use multiple AI clients (or just hate repeating yourself), you can create one config file that the MCP server reads automatically. Then your AI client configs don't need any credentials at all.

**macOS / Linux:**

```bash
mkdir -p ~/.config/fluentcart-mcp
```

```json title="~/.config/fluentcart-mcp/config.json"
{
  "url": "https://your-store.com",
  "username": "admin",
  "appPassword": "aBcD eFgH iJkL mNoP qRsT uVwX"
}
```

**Windows:**

```json title="%APPDATA%\fluentcart-mcp\config.json"
{
  "url": "https://your-store.com",
  "username": "admin",
  "appPassword": "aBcD eFgH iJkL mNoP qRsT uVwX"
}
```

Then your AI client config simplifies to:

```json
{
  "mcpServers": {
    "fluentcart": {
      "command": "npx",
      "args": ["-y", "fluentcart-mcp"]
    }
  }
}
```

<Callout type="info" title="Priority">
Environment variables beat the config file. If you set `FLUENTCART_URL` as an env var and also have `url` in the config file, the env var wins. I check env vars first — deliberately.
</Callout>

## Interactive Setup Wizard

Don't fancy editing JSON files by hand? Fair enough. The setup wizard walks you through it:

```bash
npx fluentcart-mcp setup
```

It asks three questions — your WordPress URL, username, and Application Password — then tests the connection against your site before saving anything. If the credentials are wrong, it tells you why and offers to retry. If they work, it writes the config file automatically.

```
┌  fluentcart-mcp setup
│
◇  WordPress URL
│  https://your-store.com
│
◇  WordPress username
│  admin
│
◇  Application Password
│  ••••••••••••••••••••••••
│
◇  Connected to Your Store Name
│
◇  Config written to ~/.config/fluentcart-mcp/config.json
│
└  You're all set. Run fluentcart-mcp to start the server.
```

After that, your AI client configs don't need any credentials — the MCP server reads them from the config file. One wizard run, every client covered.

<Callout type="info" title="When to use the wizard">
The wizard creates the same `~/.config/fluentcart-mcp/config.json` file described above. The only difference is that it validates your credentials before saving, so you find out immediately if something's wrong — rather than discovering it mid-conversation when your AI tries to list orders and gets a 401 instead.
</Callout>

## Multiple Stores

Managing more than one store? Create separate MCP server entries:

```json
{
  "mcpServers": {
    "store-production": {
      "command": "npx",
      "args": ["-y", "fluentcart-mcp"],
      "env": {
        "FLUENTCART_URL": "https://shop.example.com",
        "FLUENTCART_USERNAME": "admin",
        "FLUENTCART_APP_PASSWORD": "xxxx xxxx xxxx xxxx"
      }
    },
    "store-staging": {
      "command": "npx",
      "args": ["-y", "fluentcart-mcp"],
      "env": {
        "FLUENTCART_URL": "https://staging.example.com",
        "FLUENTCART_USERNAME": "admin",
        "FLUENTCART_APP_PASSWORD": "yyyy yyyy yyyy yyyy"
      }
    }
  }
}
```

Just tell the AI which store you're talking about: "Check orders on production" or "Create a test product on staging." It's smart enough to figure out which connection to use.
