### v1.1.5

*Released 6 March 2026* ‚Äî [GitHub Release](https://github.com/vcode-sh/fchub-plugins/releases/tag/fchub-multi-currency/v1.1.5)

The price projection engine was architecturally deaf. Every dynamic price update ‚Äî cart changes, variant switches, modal opens ‚Äî was broken. Hat tip to [@ManniGH](https://github.com/ManniGH) for the bug reports that exposed the full extent of the carnage.

#### Critical

- **Price projection engine was deaf to FluentCart** ‚Äî all event listeners were attached to `document`, but FluentCart dispatches every event on `window`. Custom events don't bubble across event targets. The projection engine never received cart updates, variant changes, or modal opens. Every dynamic price update was broken in production. Fixed by moving all listeners to `window`, where FluentCart has been shouting into the void this whole time.

#### High

- **Variant switching left prices in base currency** ‚Äî missing `fluentCartSingleProductVariationChanged` listener. The engine simply didn't know variants existed. Now listens and re-projects with a 200ms delay for FluentCart to finish rendering.
- **"View Options" modal prices stuck in base currency** ‚Äî missing `fluentCartSingleProductModalOpened` listener. Same problem, same fix, same facepalm.
- **Checkout subscription renewal line not converted** ‚Äî `.fct_item_payment_info` wasn't in the price selector list. It is now. Subscriptions showing "$9.99/month" next to "‚Ç¨8.72 total" was not the vibe.
- **Subscription text destroyed during projection** ‚Äî "per month, until cancel" was getting obliterated because the projection replaced entire text nodes. Now uses regex to surgically replace only the price portion, leaving the suffix text intact.
- **Pricing table elements destroyed** ‚Äî `<sup>` currency signs and `<span class="repeat-interval">` were being vaporised by `textContent` replacement. Now detects mixed content and modifies only the text node containing the price, because destroying DOM elements is not a currency conversion strategy.
- **`fchub_mc_format_price()` fatal error** ‚Äî `$optionStore` was undefined when the currency context was already cached. Every call after the first page load would crash. The function that formats prices couldn't format prices. Brilliant.

#### Security

- **Rate limiting added to public POST `/context`** ‚Äî unauthenticated endpoint could trigger unlimited FluentCRM API calls and DB writes. Now limited to 30 requests/minute per IP. Your store is no longer a free DDoS relay.
- **Provider names removed from public `/rates` response** ‚Äî was leaking which exchange rate API the store uses. Nobody needs to know that.
- **Checkout disclosure text now sanitised on output** ‚Äî admin-configurable template now runs through `wp_kses()` because trusting raw HTML from a settings field is how XSS happens.

#### Features

- **SVG flag icons** ‚Äî bundled 50 SVG country flags from flag-icons (MIT licensed). Windows users no longer see mysterious two-letter codes where flags should be.
- **Shortcode `label` attribute** ‚Äî `[fchub_currency_switcher label="Select currency:"]` renders a text label next to the dropdown. For those who believe in accessibility, or just labelling things.
- **Shortcode `align` attribute** ‚Äî `[fchub_currency_switcher align="right"]` supports right and centre alignment.
- **Per-currency decimal/thousand separators** ‚Äî display currencies can now have explicit separator config instead of the position-based heuristic that thought all right-positioned currencies use commas.

#### Code Quality

- **Resolver chain cached** ‚Äî `buildResolverChain()` no longer creates fresh objects on every call. Object creation is not a hobby.
- **Duplicated `isAllowedCurrency()` extracted to trait** ‚Äî was copy-pasted across 4 resolver classes. DRY exists for a reason.
- **Race condition in stale lock handling fixed** ‚Äî `delete_option` + `add_option` replaced with atomic `update_option`. TOCTOU bugs: the gift that keeps on giving.
- **`current_time('mysql')` replaced** ‚Äî deprecated WordPress function replaced with `wp_date()` in 2 more files. WordPress has been asking nicely since 5.3.

### v1.1.4

*Released 6 March 2026* ‚Äî [GitHub Release](https://github.com/vcode-sh/fchub-plugins/releases/tag/fchub-multi-currency/v1.1.4)

Twelve bugs walked into the codebase. None of them survived the code review.

- **EUR prices now actually look like EUR prices** ‚Äî `formatNumber()` was borrowing the base currency's decimal and thousand separators when formatting display currency prices. EUR showing as `1,234.56‚Ç¨` instead of `1.234,56‚Ç¨`. Every right-positioned currency was wrong. Every single one. Fixed.
- **`half_down` rounding no longer lies** ‚Äî the `Math.round(x * (1-EPSILON))` trick doesn't hold for all values, as it turns out mathematics hasn't changed. Replaced with a proper floor-based comparison that behaves correctly instead of mostly correctly.
- **`fchub_mc_format_price()` stops rebuilding the universe on every call** ‚Äî the function was spinning up the full resolver chain plus DB queries each time it was invoked. It now reuses the cached context like a reasonable piece of software.
- **Stale rate warning stops interrogating the database on every admin page load** ‚Äî now cached via transient with a 5-minute TTL. Your DB server sends its regards.
- **`ExchangeRate::from()` no longer throws a fatal on unknown providers** ‚Äî if a provider string in the DB didn't match the enum, you got a crash. It now falls back to Manual and carries on with its life.
- **Currency switcher `fetch()` now handles network failures** ‚Äî previously had no `.catch()`, so a network error would silently kill the loading spinner forever. Errors are now caught and surfaced like adults.
- **Rate refresh lock fixed** ‚Äî TOCTOU race condition on `get_option`/`add_option`. Now uses atomic `add_option` first, so two concurrent refreshes can't both think they won the lock.
- **Currency symbol no longer sanitized with `wp_kses_post`** ‚Äî allows HTML. A currency symbol field. Tightened to `sanitize_text_field` because `‚Ç¨` is not a `<script>` tag.
- **`current_time('timestamp')` replaced with `time()`** ‚Äî deprecated across 3 files. WordPress has been politely suggesting this for years.
- **FluentCrmSync and FluentCommunitySync now go through OptionStore** ‚Äî they were calling settings directly, bypassing the default settings merge. Fixed.
- **Rounding precision fallback aligned** ‚Äî `FrontendModule` said `0`, `Constants` said `2`. They now agree on `2`, which is the number of decimal places a price should have.
- **Sortable drag-and-drop DOM restoration fixed** ‚Äî the logic for restoring DOM order during downward drags was inverted. Moving a currency down would put it up. Philosophy aside, this is not ideal.
- **Admin sidebar `requestAnimationFrame` loop bounded** ‚Äî the injection would keep looping forever if FluentCart's markup changed. Now it gives up gracefully instead of melting the browser tab.

### v1.1.3

*Released 6 March 2026* ‚Äî [GitHub Release](https://github.com/vcode-sh/fchub-plugins/releases/tag/fchub-multi-currency/v1.1.3)

Three bugs walked into a price projector. None of them walked out.

- **Strikethrough prices restored** ‚Äî the projection engine was cheerfully nuking `<del>` tags when converting compare prices, turning "~~$100.00~~ $80.00" into just "‚Ç¨74.40" with no visual hint that it used to cost more. `findPriceTarget()` now recognises `<del>` elements, so your sale prices actually look like sale prices again.
- **"From" prices on shop cards now convert** ‚Äî "From $80.00" on product cards was silently ignored because the regex couldn't handle a currency symbol wedged between the prefix text and the digits. Affected every left-positioned currency (USD, GBP, JPY‚Ä¶) ‚Äî which is, you know, most of them.
- **Double-conversion edge case fixed** ‚Äî compare prices inside variant containers could get converted twice when the display currency used a right-positioned symbol (e.g. `93.00‚Ç¨` ‚Üí `parseFloat` would happily parse it again). Child compare-price elements are now marked as projected after the parent variant processes them.

### v1.1.2

*Released 6 March 2026* ‚Äî [GitHub Release](https://github.com/vcode-sh/fchub-plugins/releases/tag/fchub-multi-currency/v1.1.2)

Fixed exchange rate cron going missing after deactivation/reactivation cycles. The refresh event was only scheduled inside `register_activation_hook` ‚Äî if WordPress cleared it for any reason, rates would go stale with no way to recover. The `init` hook now re-schedules the cron automatically when it detects it's missing.

### v1.1.1

*Released 6 March 2026* ‚Äî [GitHub Release](https://github.com/vcode-sh/fchub-plugins/releases/tag/fchub-multi-currency/v1.1.1)

Fixed a fatal error when multiple FCHub plugins are active at the same time. The shared `GitHubUpdater` class used a `class_exists` guard that PHP's OPcache cheerfully ignored during early class binding ‚Äî so the second plugin to load would redeclare the class and take the site down. Wrapped the class inside the conditional so OPcache actually respects it.

#### Core Features

- **Currency switcher widget** ‚Äî custom dropdown with flag emojis, ARIA listbox roles, keyboard navigation, and a rate freshness badge. Place it anywhere with `[fchub_currency_switcher]`
- **Price projection engine** ‚Äî JavaScript-based real-time price conversion across all FluentCart elements. Product cards, cart drawer, checkout totals, pricing tables, variant buttons ‚Äî everything gets projected
- **Exchange rate management** ‚Äî four providers (Exchange Rate API, Open Exchange Rates, European Central Bank, Manual), automatic cron refresh, rate history with 90-day retention, staleness detection
- **Checkout disclosure** ‚Äî configurable notice at checkout with template tokens (`{base_currency}`, `{display_currency}`, `{rate}`). Injected into checkout summary, cart drawer, and cart page
- **Order snapshots** ‚Äî display currency, base currency, and exchange rate saved to order metadata on payment
- **Context resolution chain** ‚Äî URL parameter ‚Üí user meta ‚Üí cookie ‚Üí geolocation (feature-flagged) ‚Üí fallback default. First match wins, all validated against display currency whitelist

#### Integrations

- **FluentCRM sync** ‚Äî automatic contact tagging with currency tags, custom field updates on context switch and order payment
- **FluentCommunity sync** ‚Äî preferred currency written to community user meta
- **FluentCart addon** ‚Äî appears in FluentCart's integration modules, extends store settings API, injects checkout data fragments

#### Admin

- **Six-tab settings SPA** ‚Äî General, Currencies, Exchange Rates, Checkout, CRM, and Diagnostics. Built as a Vue 3 component inside FluentCart's admin
- **Real-time diagnostics** ‚Äî plugin version, PHP/bcmath status, FluentCart/FluentCRM presence, rate count, stale currency detection, feature flags
- **Stale rate admin notice** ‚Äî dashboard warning when any exchange rate exceeds the stale threshold

#### Developer

- **REST API** ‚Äî public endpoints for context get/set and rate listing; admin endpoints for settings, rate refresh, currency catalogue, and diagnostics
- **PHP API** ‚Äî `fchub_mc_format_price()` function for theme/plugin developers
- **JavaScript API** ‚Äî `window.fchubMcSwitchCurrency()`, `window.fchubMcInitSwitchers()`, `window.fchubMcProjectPrices()` globals
- **Custom hooks** ‚Äî `fchub_mc/context_switched`, `fchub_mc/rates_refreshed`, `fchub_mc/context` filter, `fchub_mc/modules` filter
- **GDPR compliant** ‚Äî personal data exporter and eraser registered with WordPress privacy tools
- **Feature flags** ‚Äî `js_projection` and `geo_resolver` for gradual rollout control
