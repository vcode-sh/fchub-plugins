- Videos delete from post AND Cloudflare when you remove them. No leftovers. Clean slate.
</div>

**Code Quality Improvements**

<div className="pl-4 border-l-2 border-fd-border space-y-2">

### <AlertTriangle className="size-4 inline mr-2" />Technical Details

<Callout type="info">
**Bug Discovery:**

Reported by [Jacob Windham](https://github.com/vcode-sh/fchub-plugins/issues). Videos disappearing after editing posts. Thank you for actually testing this and reporting it instead of silently suffering.

**What Was Broken:**

- Edit post with video → save → video disappeared. `isset($var)` when `$var = null` returns false. I thought user didn't send media. They sent null.
- Remove video from post → save → video stayed in post AND Cloudflare. Hook #1 deleted it. Hook #2 saw "no media in request" and restored existing video. Teamwork at its finest.

**The Fix:**

- `array_key_exists()` checks if key exists regardless of value. `null` is a value. Not the same as missing key.
- Global flag between hooks: "Hey Hook #2, Hook #1 already deleted this video. Don't restore it." Communication works.
- Cloudflare API cleanup verified. DELETE returns 200. Video gone. Database updated. Works as intended.

**Testing Results:**

- Test 1 (Edit without changing video): ✅ Video stays
- Test 2 (Replace video): ⏸️ Skipped (frontend issue, not backend)
- Test 3 (Delete video): ✅ Video deletes from post + Cloudflare
- Test 4 (Create new post): ✅ Works
- Test 5 (Edit post without video): ✅ No errors
- Test 6 (Multiple edits): ✅ Video survives 3+ edits

Built because editing posts shouldn't require sacrificing videos to the WordPress gods.
</Callout>

## <Package className="size-4 inline mr-2" />v0.1.0 <Badge variant="outline" className="ml-2">Beta</Badge><span className="ml-2 text-sm text-fd-muted-foreground">2025-11-05</span>

Videos work after encoding. No 404 mystery. No refresh gymnastics.

### <Bug className="size-4 inline mr-2" />Fixed

**Videos Actually Work After Encoding (No More 404 Mystery)**

<div className="pl-4 border-l-2 border-fd-border space-y-2 mt-4">
- Fixed "Video not found" errors after encoding finished
- Fixed encoding screen flashing on every page refresh
- Cloudflare said "ready!" at 40% encoding. I believed them. Mistake.
- Now I verify videos are 100% done AND files accessible before showing player
- Encoding → video works automatically. Zero refresh. Zero 404 errors.
</div>

**What Was Broken:**
<div className="pl-4 border-l-2 border-fd-border space-y-2 mt-2">
- Cloudflare webhook: `pctComplete: 40%` → rendered iframe → manifest 404
- Webhooks "best effort" → many never arrived → database stuck at pending forever
- Every refresh: encoding flash → polling → iframe (annoying)
</div>

**The Fix:**
<div className="pl-4 border-l-2 border-fd-border space-y-2 mt-2">
- Wait for `pctComplete: 100%` (not 40%)
- Probe manifest URLs before rendering (HEAD request, only render on HTTP 200)
- Update database when video ready (no more encoding flash on refresh)
</div>

Built because "just refresh the page" isn't user experience. It's surrender.

<div className="pl-4 border-l-2 border-fd-border space-y-2">
- Fixed 401 Unauthorized errors during long video encoding (15+ minutes)
- Made nonce verification flexible for status checks: invalid nonce no longer blocks polling if user is logged in

<div className="pl-4 border-l-2 border-fd-border space-y-2">
- Polling now continues after 401/403 errors instead of stopping immediately
- Frontend automatically refreshes nonce from window settings on each API call
- Added retry logic: stops polling only after 5 consecutive errors (was stopping on first error)
- Improved error handling: logs errors but continues polling unless too many failures occur

### <AlertTriangle className="size-4 inline mr-2" />Important Note

<Callout type="info">
**Nonce Expiration During Long Polling**

WordPress nonces can expire during long encoding sessions (15+ minutes). The plugin now handles this gracefully by allowing requests with expired nonce if user is still logged in. Warnings are logged for monitoring but don't block functionality. This ensures videos aren't stuck in "encoding" state when Cloudflare finishes encoding.
</Callout>

### Before updating
- Read the changelog (you're already here, good)
- Check "What Broke" section (if it exists)
- Backup database if it's a major version
- Test on staging if you have one (you should)

### How updates work
1. New version released on [GitHub Releases](https://github.com/vcode-sh/fchub-plugins/releases)
2. Download latest `fchub-stream.zip`
3. Deactivate old version, delete it
4. Upload new version, activate

### After updating
- Test video uploads
- Check existing videos still play
- Verify settings didn't reset
- Report bugs if something broke
